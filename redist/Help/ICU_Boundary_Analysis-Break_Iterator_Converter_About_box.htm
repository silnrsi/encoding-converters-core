ICU Boundary Analysis/Break Iterator

The International Components for Unicode (ICU) provides several processes for determining character, word, line-break, and sentence boundaries. The implementation in the EncConverters' wrapper on ICU Boundary Analysis/Break Iterator is for doing word boundary analysis, and you can use this transducer to break up, for example, a long string of Thai text and spaces will be inserted between word forms.

See http://userguide.icu-project.org/boundaryanalysis for more details on the capabilities for this ICU transducer.

Saving the converter in the System Repository:

By default, if you click the Apply or OK buttons, the configured converter will be returned to the client application as a temporary converter for temporary use. Once the client application (e.g. FieldWorks or Word) is closed or releases the converter, it will no longer be available. If you want the converter to be permanently available to client applications, then you must explicitly add it to the System Repository using the Save in System Repository button.

When you click this button, the following dialog box will be displayed to query for a friendly name by which the converter will be known in client applications:

 

For the ICU Boundary Analysis/Break Iterator, the default name will be Boundary Analysis/Break Iterator. However, you can change it to something more meaningful if desired (e.g. "Thai word-breaks").
