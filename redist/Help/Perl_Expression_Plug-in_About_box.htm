Perl Expression Converter

This converter allows you to write a Perl expression to do text processing. For example, you could have the following Perl expression to reverse a string of characters:

 

$strOut = reverse($strIn);

 

Or this expression to convert the input string to lower case:

 

$strOut = lc($strIn);

 

Notice that the input data to be converted come into the Perl expression via the $strIn variable. The output data must be assigned to the $strOut variable.

Alternately, you can use a single scalar variable for both input and output: $strInOut. This is useful for doing search and replace regular expressions. For example, the following expression can be used to convert the input string using the rot13 algorithm:

 

$strInOut =~ y/A-Za-z/(N-ZA-Mn-za-m)/;

 

The SILConverters' Perl Expression converter requires a separately downloaded Perl distribution (currently, version 5.10.1) to be installed on your system to operate correctly. You can download a free Perl distribution, such as ActiveState’s Perl v5.10.1 from this website: Active State (or perhaps in their previous downloads area).

Setup Tab

Here is the configuration setup tab for the Perl Expression EncConverter:

Figure 1: Perl Expression Setup [1], [2], [3], [4], [5], [6]

 

1.      This area is for entering your Perl expression.

2.      This area is for indicating which encoding type your expression expects and returns. If your data is Unicode-encoded, be sure to select that option or your data may be incorrectly converted. For Non-Unicode (byte) data, the default system code page will be used to convert your data when necessary.

After configuring these two items, you can click the Apply button to accept the expression.

The Setup tab also has the following options:

3.      The Previous Expressions combo box includes a few example Perl expressions and remembers any new ones you add. The Delete button can be used to remove unwanted expressions.

4.      The Distro Config button allows you to set up the path to your Perl Distribution's library folders (e.g. C:\Perl, C:\Perl\lib, C:\Perl\site\lib) as well as to specify certain Perl modules to be automatically loaded for all expressions (e.g. Win32, or SIL::RTF::Unicode). If you get an error message saying that some *.pm file can't be found, then you probably don't have the correct path to the file configured.

1.      If you want the expression to be permanently added to the System Repository, then you must click the Save in System Repository button (see below).

2.      You can click on the Test Area tab in order to test the converter with some sample data.

You may not see script errors until you actually use the converter to convert some data. So for this converter type in particular, it is important to test it before trying it with real data.

Saving the converter in the System Repository:

By default, if you click the Apply or OK buttons, the configured converter will be returned to the client application as a temporary converter for temporary use. Once the client application (e.g. FieldWorks or Word) is closed or releases the converter, it will no longer be available. If you want the converter to be permanently available to client applications, then you must explicitly add it to the System Repository using the Save in System Repository button.

When you click this button, the following dialog box will be displayed to query for a friendly name by which the converter will be known in client applications:

 

For the Perl Expression converter, the default name will be the Perl expression itself. However, it is a good idea to make it more meaningful (e.g. Convert English consonants into 'C' and vowels into 'V').

The Advanced... button can be used to enter further, optional information about this converter which is also put into the System Repository and can be used by various client applications. Click on this button to bring up the following dialog box:

 

Though these values are not necessary for the operation of the converter, they can be helpful to various client applications. For example, the Clipboard EncConverter can be configured to filter the list of displayed converters based on the Encoding Name and/or the Transduction Type configured here.

More Sample Expressions

# turn all non-vowels into 'C' and vowels into 'V'

$strInOut =~ s/[^aeiou ]/C/g;

$strInOut =~ s/[aeiou]/V/g;

 

# similar expression for Devanagari range of Unicode

# first convert all vowel characters to ‘V’

# then convert all consonants to ‘C’

$strInOut =~ s/[\x{0905}-\x{0914}\x{093E}-\x{094c}]/V/g;

$strInOut =~ s/\p{Devanagari}/C/g;

# NOTE: if your data is Unicode-encoded, then you

# must indicate via the radio buttons on the Setup

# tab that your Perl script expects and returns a

# "Unicode String" (these options may not be visible

# in all client applications)

 

# here’s an expression to turn a book/chapter/verse

#  reference into the “vref” SFM field syntax.

# the input is something like “Rut 1:1”

#  the output will be “Rut01001”.

# note that if the match is not found, you’ll want

#  the input string returned as the output variable.

# also note that you can use “sprint”

#  to set the output variable.

if( $strIn =~ m/(.*?)(\D{3}) (\d):(\d+)(.*)/ )

{

    $strOut = sprintf "$1$2%02d%03d$5", $3, $4;

}

else

{

    $strOut = $strIn;

}

 

# here's an expression to fix bad comma placment

# (e.g. /कहा ,"मैं…"/ becomes /कहा, "मैं…"/ and

# /कहा , "मैं…"/ becomes /कहा, "मैं…"/)

$strInOut =~ s/ ,([^ ])/, $1/g;

$strInOut =~ s/ , /, /g;
