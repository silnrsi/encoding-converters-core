Bing Translator

This converter allows you to access the same translation system as used by the Bing Translator™ to do text processing. The Azure Translator is a cloud-based neural machine translation service that is part of the Azure Cognitive Services family of REST APIs. It can be used to perform language translation and other language-related operations between a growing number of languages (see this website for specific languages supported). Within EncConverter-enabled applications, it requires a live internet connection to function.

See this website for more information on Azure Translator.

Setup Tab

Here is the configuration setup tab for the Bing Translator EncConverter:

Figure 1: Bing Translator Setup 1, 2, 3, 4, 5, 6, 7, 8

 

1)      This area is for choosing the operation you want the converter to do. There are 4 options:

a)      Text Translation – Supports text translation between supported Source and Target languages.

b)      Transliterate – Supports the transliteration of characters of the selected Source language to the corresponding characters or letters of the selected Target language script. Note: not all Source languages support Transliteration. See this table to know whether a certain combination is supported. The Source script (4) and Target script (5) boxes will be empty if Transliteration isn’t supported for the selected Source language (2).

c)      Translate + Transliterate – Supports transliterating the result of a translation pass. Note: not all languages that support Translation also support Transliteration. The Source script (4) and Target script (5) boxes will be empty if Transliteration isn’t supported for the selected Target language (3).

d)      Dictionary Lookup – While the Translation options work better if entire sentences or paragraphs are processed in a single call (rather than word-by-word, which may not give the desire order for the translated words in the target language), this option is for getting one or more translations of a single word in the Source language. See the Dictionary column in the table on this webpage to know whether a given Source language supports this option.

Note: if there are multiple words in the input text, the lookup will only return data for the first word in the input. If there are multiple translations for the given word, then the converted text will be in the Ample ambiguity form (i.e. "%<count=n>%<translation_1>%<translation_2>%...%<translation_n>%", e.g. "%2%from%with%").

2)      This combo box is for selecting the Source language to use for the operations mentioned above (1). Note: the default value for this field is Auto-detect, which allows you to use the same converter for multiple Source languages translating into the same Target language. But note that not all Source languages support auto-detection. See the Auto Language Detection column of the table on this webpage to know whether a particular language supports auto-detection.

3)      This combo box is for selecting the Target language the input text should be translated into. It is required for all but the Transliterate option (1)b).

4)      This text box is for displaying the Source script used in the Transliterate options.

For the Translate + Transliteration option (1)c), the Source script is fixed as the default script of the Target language from which you can choose to see the results in another script by selecting it in the Target script combo box (currently, most Target languages only support Latin).

For the Transliteration option (1)b), this text box will show the option(s) available based on what Target script is selected. Note: currently, most Transliteration options are either from the default script for the selected Target language to Latin (Latn) or vice-versa. That is, you can also set up a converter that will Transliterate from a Latin script representation of a given Source language, into its default script as well.

5)      This combo box is for selecting the script that you would like to see the results in after conversion. This is only used for one of the two Transliteration options:

For the Translate + Transliteration option (1)c), the Target script is (currently) limited to Latin (Latn).

For the Transliteration option (1)b), all transliteration scripts possible are listed in the combo box to select from. This allows the default option of transliterating from the default script of the given Source language to Latin or from the Latin representation of the Source language back into the default script for the Source language.

6)      This button is for entering your own API key for the Azure Translator resource. See the Prerequisites section on this webpage, for how to get your own Azure account and create a Translator resource from which to obtain your own Translator API key and location to use for this transducer.

Note the default API key used by EncConverters-enabled applications is using the free pricing tier, which is limited to 2 million characters/month for all users. Once the 2 million characters/month are used up, it will begin to return an error message indicating that the resource is now used up for the month. See the section below (“Creating your own Azure Translator resource”) for more information on creating your own Translator resource.

7)      If you want the converter to be permanently added to the System Repository, then you must click the Save in System Repository button (see below).

8)      You can click on the Test Area tab to test the converter with some sample data.

You may not see script errors until you use the converter to convert some data. So, for this converter type in particular, it is important to test it before trying it with real data.

Saving the converter in the System Repository:

By default, if you click the Apply or OK buttons, the configured converter will be returned to the client application as a temporary converter for temporary use. Once the client application (e.g. FieldWorks or Word) is closed or releases the converter, it will no longer be available. If you want the converter to be permanently available to client applications, then you must explicitly add it to the System Repository using the Save in System Repository button.
