Google Translate

This converter allows you to use Google Translate™ to do text processing within EncConverter-enabled applications. Google Translate is a cloud-based translation service. It can be used to perform language translation and other language-related operations between a growing number of languages (see this website for specific languages supported). Note that it requires a live internet connection to function.

Google Translate available within EncConverters-enabled applications is limited to 500K characters/month for all users. If you need more data for your purposes, you can create your own (free or paid) Google Cloud plan and enter the Translate PI credentials in the configuration dialog shown below.

See this website for more information on getting your own Google Cloud Translate API service.

Setup Tab

Here is the configuration setup tab for the Google Translate EncConverter:

Figure 1: Google Translate Setup 1, 2, 3, 4, 5

 

1)      This combo box is for selecting the Source language of the input text for the translation operation. Note: the default value for this field is Auto-detect, which allows you to use the same converter for multiple Source languages translating into the same Target language. But note that not all Source languages support auto-detection.

2)      This combo box is for selecting the Target language the input text should be translated into.

3)      This button is for entering your own API key for the Google Translate resource. See this webpage for how to get your own Google Cloud service account key to use for this transducer.

Note the default API key used by EncConverters-enabled applications is using the free pricing tier, which is limited to 500K characters/month for all users. Once the 500K characters/month are used up, it will begin to return an error message indicating that the resource is now used up for the month.

4)      If you want the converter to be permanently added to the System Repository, then you must click the Save in System Repository button (see below).

5)      You can click on the Test Area tab to test the converter with some sample data.

You may not see script errors until you use the converter to convert some data. So, for this converter type in particular, it is important to test it before trying it with real data.

Entering your own Google Translate API key:

Once you acquire your own API key (a json message) for a Google Cloud Service Account from the Google website,  you can enter it using the dialog that appears by clicking the button on the Setup dialog (3)  above. Enter the json message you receive from Google in the Enter Google Translate Credentials (json) field shown below:

 

Figure 2: Enter Google Translate API Credentials Dialog

 

Using Environment Variables to Store Google Translate Credentials

Note that when the translator is used in different client applications (e.g. Paratext, Word, or the Clipboard EncConverter), the settings (including the credentials json message mentioned above) are stored (encrypted) in the application settings file of the specific application using it. So if you have your own credentials, you must enter it in each application that you want to use it in by editing the converter and setting the json message in the Setup tab as described above. Alternately, you can add the path to the downloaded json file to the following environment variable (GOOGLE_APPLICATION_CREDENTIALS), and it will be available to all applications:

Figure 3: Environment Variable for Google Translate Credentials

 

Saving the converter in the System Repository:

By default, if you click the Apply or OK buttons, the configured converter will be returned to the client application as a temporary converter for temporary use. Once the client application (e.g. FieldWorks or Word) is closed or releases the converter, it will no longer be available. If you want the converter to be permanently available to client applications, then you must explicitly add it to the System Repository using the Save in System Repository button.

When you click this button, the following dialog box will be displayed to query for a friendly name by which the converter will be known in client applications:

Figure 4: Enter Converter Friendly Name Dialog
